Generative Line Art

Midterm Project

Design

For the lines, we wanted them to appear to make a 3D shape, so through trial and error we came up with an algorithm that made them move randomly and fake a 3D shape. Although this doesn't always work perfectly, the best examples of this are in the Examples Page.

Another important consideration was what drawing library to use. We ended up using Two.js because it was simple and easy to pick up. Since there was not much time to make the project, we wanted to spend as little time as possible learning the technologies.

We wanted to make it really easy to customize how the art turned out, so we emphasized having as many different controls as possible. dat.gui did a lot of the work for us, so we could just take the value stored in the controls and have that be a variable in the JS that generates the lines.

Challenges

One of the main challenges of the project was coming up with the correct math to draw the lines so that they didn't overlap and looked the way we wanted them to. We ended up having to use some algebra to calculate the distance between the lines using their slopes.

Another challenge was animating the lines, originally we wanted to animate them so they were constantly moving. This ended up being too difficult so we only animated them as they were being drawn.

A smaller challenge was having the art download correctly, when we tried to export it the file would break. After some debugging, we realized the issue was with the meta data we put in. When the meta data was fixed the file could be viewed correctly.
