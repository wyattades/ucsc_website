Todo List App

Week 6

Design

The design is the same as the previous week's, but I had to add some styling for the TodoList items.

Also, I decided to make #input a form element so it would capture ENTER key presses as well as clicking submit.

Challenges

I was mainly challenged by the fact that we were expected to use innerHtml to set the contents of #output. If this were the case, I wouldn't know how to capture click events on the remove and toggle buttons on each todolist item. Instead, I decided to create all elements with document.createElement() so I could add event listeners easily. This was a little more verbose, but fixed my problem and is actually much more computationaly efficient than using innerHtml.

Another big challenge was not using jQuery, which probably would have made my life easier.
