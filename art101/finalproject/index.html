Generative Poly Art

Final Project

Design

We wanted to build off the idea of our midterm but (literally) add another dimension by making the art generate triangles instead of lines. This would make it easier to get the illusion of 3D shapes that we initially wanted.

The first version we did was the ‘tunnel’ version, where triangles are drawn successively inside of each other with the color getting darker. This worked well, but there were not many things that we could create custom controls for.

The second version is the ‘grid’ version. There is a grid of points that are the basis for triangles. The points are then moved around a random value to make the triangles look 3D. The colors of the triangles are selected based on one color the user picks and a range of colors generated using linear interpolation.

The GUI is the same base as the midterm, just with different options that the user can change for each version (‘tunnel’ vs. ‘grid’).

Challenges

One challenge was getting the correct color values for the triangles. The way we decided to change the colors was based on HSL color codes, but Two.js required the colors to be Hex codes, so we had to convert between the different systems frequently. Color was also difficult because we ended up deciding to use linear interpolation to generate the colors, which was difficult to wrap our minds around and to get to look aesthetically pleasing.

Another challenge was not having gaps between the triangles in the grid. Initially, we could make the grid but there was black space between the shapes. We never figured out why there was initially the spaces, but we fixed it by adding a stroke to the outside of the shapes that was either the same color as the shape or the color the user picked.
